# Frontend Quick Check

변경 후 바로 실행해서 확인할 수 있는 최소 체크리스트입니다.

1) 의존성 설치/복구
- `cd frontend`
- `npm install`

2) 정적 검사(가능하면)
- `npm run lint`
- 실패 시 `node_modules`를 삭제 후 재설치: `rm -rf node_modules package-lock.json && npm install`

3) 빌드 테스트
- `npm run build`

4) 화면 실행 확인
- 백엔드가 실행 중이어야 모의 주문/차트/로그인 API 연결이 정상 동작합니다.
- `npm run dev`
- 브라우저에서 `http://localhost:3000` 접속
- 변경사항이 반영되지 않으면 브라우저 캐시 새로고침 + 컨테이너 재기동:
  - `docker compose restart frontend`
  - 이후 브라우저 강력 새로고침 (`Cmd/Ctrl + Shift + R`)

5) 모의 투자 화면 1회 점검 포인트
- 헤더: `모의 투자 주문`, `현재가 갱신`, `데모 계좌 초기화`
- 모달: 티커/수량/단가 입력, 수수료 반영, 주문 버튼 비활성화 조건, 오류 메시지 노출
- 보유 종목: 수익률/평가금액/매도 버튼
- 체결 이력: 최신 12건 표시 여부

6) 백테스팅 1회 점검 포인트
- 메뉴에서 `백테스팅` 진입 가능성 확인 (`X` 로고 아래 또는 사이드바 메뉴)
- 티커(예: AAPL), 기간, MA 파라미터 입력 후 `백테스트 실행`
- 프리셋(예: 올웨더/성장형/보수형)을 눌러도 `백테스트 실행`이 동작하는지 확인
- 종목 검색 박스에서 S&P 500 기반으로 가져온 후보 목록이 출력되는지 확인
- 초기 자금 프리셋 버튼(5만/10만/30만/100만)이 입력값에 반영되는지 확인
- 결과 카드: 최종 자산/총 수익률/MDD/승률/거래 수
- 실행 중 진행 단계 텍스트 및 진행바가 표시되는지 확인 (요청 지연 시에도 상태 피드백이 유지되는지)
- 자산 곡선 및 체결 내역 표시

문제가 계속되면 다음 2가지만 바로 보내주세요.
- 터미널 에러 로그
- 브라우저 콘솔 에러 로그
